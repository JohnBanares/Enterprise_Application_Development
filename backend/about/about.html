  • General description of how the application works

    • This application follows the MERN stack so the this project is all done in one language JavaScript and the technologies consist of MongoDB, Express, Node and React. And follows a 3 tier architectre system.

    Presentation Tier: React handles the UI and updates according to user inputs.

    Application Tier: Express and Node handle the communication and acts as a bridge to handle user request and server responses.

    Data Tier: Mongodb handles the storage of data in this case products

    Here are the features of this application:

    Here are the features of this application:

    Home Page:

    - Search database by brand name

    - Insert a new product, by creating a name, id, brand and price

    - To delete must first use the search box to find a brand. Then click the delete button to display delete icons, and choose multiple product or product to delete. It also shows how many products are selected to delete.

    - To update a product its acts the same way, you must first search a product the click the update button. This will reveal the edit button on each product container, in which you can edit and save all changes to each field.

    About Page:

    - This page fetches html data from about.html stored on the backend folder using axios and inserts this data in an empty div in the about component

  • Description of the technologies involved

    • The technologies used:

    Mongodb atlas: A cloud database serive that allows for the development of custom models and provides scalability security and monitoring.

    React: A JavaScript library used for building UI with reusable commponents, allowing for interactivity and seamless user experience

    Node javascript runtime enviroment that allows for the development of web applications and handles the development of backend APIs

    Express: A node web application framework that allows to structure api end points in the backend folder. Simplifies the development of APIs, handling request and responses on the server-side.

    Axios: Used to make creating AJAX requests easier

    React router dom: handles the rendering of components and routes for web navigation
